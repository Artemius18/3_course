промежуточное по с обработкой ошибок

Объяснение:
1. express: Импортируем Express.js и создаем экземпляр приложения (app).
2. app.get('/error', ...): Определяем маршрут /error, который генерирует ошибку случайным образом. Если выпадает случайное число больше 0.5, генерируется ошибка.
3. Обработка ошибок через next(): Если в процессе выполнения маршрута возникает ошибка, она передается функции next(err). 
4. Express автоматически перенаправляет выполнение к следующему обработчику ошибок, который принимает четыре аргумента (err, req, res, next).
5. app.use(...): Это специальный middleware, который используется для обработки всех видов ошибок, которые передаются через next().
